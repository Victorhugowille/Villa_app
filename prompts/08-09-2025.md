# Prompt: Definição Inicial do VillaBistrô

**Data:** 08/09/2025
**Contexto:** Início do projeto, saindo de testes para uma aplicação real.

## 🗣️ Prompt do Usuário
"Quero criar um aplicativo de gestão para meu restaurante chamado VillaBistrô. Vou usar Flutter no front-end e Supabase no back-end.
Preciso que ele funcione no Celular (Garçom) e no Desktop (Caixa).
Como devo organizar as pastas do projeto para manter o código limpo usando Provider?"

## 🤖 Resposta (Solução Arquitetural)

Sugiro uma arquitetura baseada em **Features** ou **Camadas** simplificadas (MVVM light).

### Estrutura de Pastas Definida:
```text
lib/
├── data/          --> Classes de Modelo (Table, Order, Product)
├── providers/     --> Lógica de Estado (AuthProvider, TableProvider)
├── screens/       --> Telas visuais (Divididas por funcionalidade)
│   ├── login/
│   ├── home/
│   └── config/
├── services/      --> Conexões externas (Impressão, API)
├── widgets/       --> Componentes reutilizáveis
└── main.dart      --> Inicialização e Configuração de Rotas
